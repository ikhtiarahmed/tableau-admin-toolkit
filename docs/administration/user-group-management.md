# Chapter 3 – User & Group Management

## Purpose

User and Group Management is one of the most important responsibilities of a Tableau Cloud Administrator. A well-designed group structure simplifies permission management, improves security, and reduces administrative overhead.



## Objectives

This chapter explains how to:

- Organize users efficiently
- Design scalable group structures
- Assign permissions consistently
- Reduce administrative effort
- Improve security



## Why Groups Matter

Instead of assigning permissions directly to individual users, permissions should be granted to groups whenever possible.

Benefits include:

- Easier administration
- Consistent permissions
- Faster onboarding
- Simpler audits
- Reduced errors



## Recommended Group Structure

Example groups:

- Executives
- Finance
- Sales
- Supply Chain
- Human Resources
- Information Technology
- Developers
- Creators
- Explorers
- Viewers
- Project Leaders
- Site Administrators



## Best Practices

- Assign permissions to groups instead of individual users.
- Review group membership regularly.
- Remove inactive users promptly.
- Keep group names consistent.
- Document the purpose of each group.



## Common Mistakes

- Granting permissions directly to individual users.
- Creating duplicate groups with similar purposes.
- Giving Creator licenses when Explorer or Viewer roles are sufficient.
- Forgetting to remove users who no longer need access.



## Administrator Checklist

- Group structure documented
- User access reviewed
- Inactive users removed
- Group naming standardized
- Permissions audited



## Summary

A well-designed group strategy makes Tableau Cloud easier to manage, improves security, and supports long-term scalability.
