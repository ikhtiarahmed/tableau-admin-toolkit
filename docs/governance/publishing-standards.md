# Publishing Standards

## Purpose

Publishing standards ensure that workbooks, dashboards, data sources, and flows published to Tableau Cloud are accurate, secure, maintainable, and easy for users to understand.

Following consistent publishing practices improves trust in analytics and reduces long-term maintenance.



# Before Publishing

Before publishing any content, verify the following:

- Dashboard has been tested.
- Filters work correctly.
- Calculations are validated.
- Performance has been reviewed.
- Data source is accurate.
- Titles and descriptions are complete.



# Workbook Requirements

Every workbook should include:

- Meaningful workbook name
- Business description
- Project assignment
- Appropriate permissions
- Owner information



# Dashboard Requirements

Every dashboard should:

- Have a clear title
- Display the last refresh date (when appropriate)
- Use consistent colors and formatting
- Avoid unnecessary sheets
- Be optimized for performance



# Data Source Requirements

Published data sources should:

- Use approved naming standards
- Include descriptions
- Identify the data owner
- Be certified when appropriate
- Minimize duplicate data sources



# Tableau Prep Flow Requirements

Flows should include:

- Clear names
- Descriptions
- Error handling where possible
- Appropriate schedules
- Owner information



# Publishing Checklist

Before publishing, confirm:

- Naming standards followed
- Performance tested
- Permissions verified
- Description added
- Owner assigned
- Sensitive data reviewed



# Common Mistakes

Avoid:

- Publishing test workbooks
- Duplicate dashboards
- Missing descriptions
- Incorrect permissions
- Publishing directly to Production without review



# Summary

Publishing standards improve consistency, governance, and user confidence while helping administrators maintain a well-organized Tableau Cloud environment.
