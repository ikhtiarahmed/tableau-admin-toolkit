# Naming Standards

## Purpose

Consistent naming standards make Tableau Cloud easier to manage, search, and maintain. Clear naming conventions also help users quickly identify the purpose, owner, and status of content.



# General Principles

Every published asset should have:

- A meaningful name
- Consistent formatting
- No unnecessary abbreviations
- Clear business context
- Standard capitalization



# Project Naming

Recommended format:

Department - Function

Examples:

- Finance
- Sales
- Human Resources
- Supply Chain
- Executive Reporting



# Workbook Naming

Recommended format:

Department - Business Process - Report Name

Examples:

- Sales - Daily Sales Dashboard
- Finance - Monthly Budget Report
- HR - Employee Headcount



# Dashboard Naming

Use names that describe the business question being answered.

Good Examples:

- Sales Performance Overview
- Inventory Health Dashboard
- Customer Order Status

Avoid:

- Dashboard 1
- Test Dashboard
- New Report



# Data Source Naming

Recommended format:

System - Subject Area

Examples:

- Snowflake - Sales
- Snowflake - Inventory
- SAP - Finance



# Tableau Prep Flow Naming

Recommended format:

Source - Process

Examples:

- Snowflake - Daily Sales ETL
- SAP - Inventory Refresh



# Best Practices

- Keep names concise.
- Avoid special characters.
- Use consistent capitalization.
- Do not include personal names unless necessary.
- Archive outdated content instead of renaming active assets.



# Summary

A standardized naming convention improves discoverability, simplifies administration, and supports long-term governance.
